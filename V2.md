# V2: README Injector Architecture

*This document outlines the planned refactor to move away from a dynamic Vercel API image and instead use a GitHub Action to directly inject clickable HTML avatars into the repository's README.*

## Why V2?
Currently, the Vercel API generates a single `.png` image. Due to GitHub's strict image proxy (`camo`), it is technically impossible to embed multiple unique links (one for each avatar) inside a single image. To make each avatar click through to the respective contributor's profile, the avatars must be rendered as raw HTML `<a>` and `<img>` tags directly inside the `README.md`.

## The Plan

### 1. Target Repository Decision
This script needs to modify a `README.md`. It must be decided whether:
- **Option A**: This script remains in a separate repository (like `top-contributors-api`) and pushes the updated README to the user's main profile repository (e.g., `username/username`). This requires setting up a Personal Access Token (`GH_TOKEN`).
- **Option B**: The `scripts/aggregate.js` and `.github/workflows/aggregate.yml` are moved directly into the user's profile repository (`username/username`). This allows the action to run natively and update its own README using the free, built-in `GITHUB_TOKEN`.

### 2. Clean Up Vercel Artifacts
Since Vercel will no longer be needed, the following will be removed:
- `api/contributors.js`
- `vercel.json`
- `local-dev.js`
- `canvas` dependency from `package.json`

### 3. README Injector Logic
Modify `scripts/aggregate.js` to:
1. Fetch and aggregate contributors.
2. Read the target `README.md`.
3. Locate injection markers, such as `<!-- START CONTRIBUTORS -->` and `<!-- END CONTRIBUTORS -->`.
4. Generate the HTML for the contributors:
   ```html
   <a href="https://github.com/username">
     <img src="avatar_url" width="80" style="border-radius: 50%; margin: 5px;" alt="username" title="username"/>
   </a>
   ```
5. Inject the HTML between the markers and save the file.

### 4. Workflow Update
Modify `.github/workflows/aggregate.yml` to:
- Run monthly.
- Commit the modified `README.md` file back to the repository instead of just caching data to the `data/` folder.
